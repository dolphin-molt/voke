# 更新日志

## 0.4.0 - 2026-07-22

### 新功能

- 顶部设备胶囊支持按住左右拖动调整顺序；自定义顺序会保存在本机，并在设备断开、重连或重启 Voke 后继续生效。
- 新增通用的 App 情景方案：手柄、外接 HID 键盘和鼠标按键都可以根据当前前台 App 自动切换到专属映射，不需要为某个软件硬编码。
- 主界面新增 `APP ROUTE` 情景条，可查看当前目标 App、绑定已有方案、复制当前方案为 App 专属方案，或随时恢复通用方案。
- 打开 Voke 配置时会保留最近一个外部目标 App，避免 Voke 自己抢到前台后导致配置对象跳变。
- 设置页新增完整的情景方案管理，可重命名、删除、选择通用回退方案、查看或清除 App 绑定。
- 情景方案导入会拦截同一设备内重复的 App 绑定，并继续兼容没有 App 字段的旧版 JSON v2 配置。
- 修正专属方案可能被误设成整机通用方案的问题：通用回退与 App 专属方案现在是明确分离的一对一关系，切到微信等其他 App 必定回到通用方案。
- ChatGPT / Codex 专属方案新增“当前 App”动作，包括新建任务、模型与推理选择器、听写、语音模式，以及可绑定快捷键的提高/降低/循环推理强度。
- App 动作改为保存 Codex command ID，并自动同步 `CODEX_HOME/keybindings.json`；用户不再需要在 Voke 重复录入快捷键。
- 使用文件系统事件监听 Codex 快捷键配置，触发时直接读取内存缓存，不增加手柄响应延迟。
- 配置路径按当前 macOS 用户与 `CODEX_HOME` 动态解析，不写死用户名或单机路径。
- 从 Voke 配置窗口直接测试 App 动作时，会先把绑定的目标 App 置前再发送快捷键；未配置快捷键的动作会在菜单中明确标注。

### 修复

- Codex“开始听写”改为真正的按住型动作：按下控制器按键时持续按住听写快捷键，松开时释放，并关闭会干扰听写的按键连发。
- 映射的 ESC、Return 及其他键盘按键现在统一作为系统级键盘事件发送，修复 Voke 不在前台时无法稳定关闭录音浮层、菜单或弹窗的问题。
- 顶部运行状态不再只看映射总开关；设备离线或缺少输入监控、辅助功能时会直接显示真实原因，避免绿色“运行中”误报。

### 验证

- 新增真实设备与权限回归记录，已完成 USB 键盘、鼠标、重启恢复、权限状态和 App 情景回退验证；手柄离线项保留为明确待验收清单。
- 自动化测试增加到 55 项，覆盖运行状态判定。

## 0.1.1 - 2026-07-21

### 修复

- 修复安装后应用图标四角被显示为黑色正方形的问题，圆角外区域现在使用透明通道。
- 安装包文件名加入版本号，避免不同版本下载后名称完全相同并触发缓存混淆。

### 发布

- 将公开测试版升级至 Build 8，并同步更新 GitHub Release 与落地页下载地址。

## 0.1.0 - 2026-07-21

### 新功能

- 用 macOS 兼容游戏手柄触发快捷键、滚动、App 切换、截图和自定义 ZSH 命令。
- 用右摇杆移动鼠标，并将手柄按键映射为鼠标点击。
- 支持 ESC、左右修饰键组合以及直接切换中英文输入源。
- 为手柄、外接 HID 键盘和鼠标提供按设备保存的独立映射。
- 应用重启后自动恢复映射，关闭窗口后继续后台运行，并可选择登录时启动。
- 提供持久诊断日志、脱敏导出和配置导入导出。
- 使用手绘设备映射面、胶囊控件和明亮、石墨、素描胶囊三套主题。

### 文档

- 新增静态产品落地页、测试 DMG 打包脚本和首次打开说明。
- 发布公开 GitHub 仓库和首个可下载测试包。

### 已知限制

- 当前 DMG 使用本地自签名，尚未经过 Apple 公证。首次打开时需要在“系统设置 → 隐私与安全性”中选择“仍要打开”。
- 不同品牌手柄和特殊 HID 设备仍需逐台验证。
